# Temporal 已知风险与应对

## 范围

本文档记录 Phase A-F 范围内已知风险、影响和建议操作。

## 风险 1：通道容量上限

- 描述：当前按 4 通道固定映射 source。
- 影响：超过 4 个 source 时，超出部分不会进入录制会话。
- 现状：已通过 `recordingSourceCount` 和 `recordingSessions` 对齐。
- 操作建议：现场多源场景需提前确认优先 source。

## 风险 2：网络抖动导致超时停录

- 描述：SST 更新延迟可能触发 inactive timeout。
- 影响：会话被提前关闭，形成多个碎片文件。
- 现状：已覆盖 timeout refresh 与重连恢复测试。
- 操作建议：高抖动网络下适当提高 timeout。

## 风险 3：远端日志可见性依赖 SSH 稳定性

- 描述：SSH 波动会影响日志拉取与状态观察。
- 影响：运维判断启动状态变慢。
- 现状：已实现轮询与失败提示。
- 操作建议：先确认 SSH 稳定，再执行流监听。

## 风险 4：磁盘空间耗尽

- 描述：长时间录制可能占满本地磁盘。
- 影响：写盘失败或文件损坏。
- 现状：暂未实现配额控制。
- 操作建议：定期清理 [recordings](recordings)，并做容量监控。

## 风险 5：配置漂移

- 描述：本地配置与远端 odaslive 配置不一致。
- 影响：端口可达但无有效数据流。
- 现状：已有模板配置，但无自动校验。
- 操作建议：联调前逐项核对主机、端口、命令。

## 回退策略

1. 停止监听并停止 odaslive。
2. 保存当前日志和录制文件样本。
3. 使用 [docs/runbook.md](docs/runbook.md) 逐步重启流程。
4. 若问题持续，优先回退到最近稳定配置与网络路径。
